# خارطة طريق مضاعفة القيمة (10x)

النقلة من «عرض مقنع لشركة AI» إلى «نظام يعتمد عليه صاحب عمل يومياً بأمواله الحقيقية».

| المرحلة | الميزة | الحالة |
|---------|--------|--------|
| أساس | **F1** سجل تدقيق ثابت (append-only) + طبقة ديمومة | ✅ سجل التدقيق · الترحيل الكامل لـ Supabase مستمر |
| أساس | **F2** فرض الأدوار على القرارات حسب المصفوفة | ✅ |
| تمييز | **F3** دراسات جدوى بذكاء LLM حقيقي (مع fallback) | ✅ |
| ثقة | **F4** قيود محاسبية مزدوجة + فاتورة ZATCA (VAT 15%) | ✅ |
| تبنّي | **F5** الملخص اليومي عبر WhatsApp/Email + إشعارات | ✅ |
| رؤية | **F6** ربط عتبة التعلّم بخط الأفكار + حلقة أتمتة كاملة | ✅ |

> ملاحظة أمانة: كل ميزة تعمل بكامل قوتها عند توفّر مفاتيحها (Supabase / OpenAI /
> مزود إشعار)، وتتدهور بأناقة إلى وضع آمن بدونها — تماماً كنمط بقية النظام.
> الترحيل الكامل لكل المتاجر من الذاكرة إلى Supabase مهمة مستمرة يقودها سجل
> التدقيق كنقطة انطلاق.

## معايير القبول (ملخص)
- **F1:** كل قرار حوكمي يُقيَّد في سجل تدقيق غير قابل للتعديل (مَن/متى/ماذا/بأي صلاحية)، قابل للاسترجاع والتصفية.
- **F2:** الاعتماد مرفوض في الـ API لمن لا يملك صلاحية الفئة (T2/T3 للمالك)، لا في الواجهة فقط.
- **F3:** الدراسة تستدعي LLM بمدخلات حقيقية وتُنتج تقريراً مُعلّلاً؛ وبلا مفتاح تعود للـ heuristic دون تعطّل.
- **F4:** كل مبلغ معتمد يُنشئ قيداً مزدوجاً متوازناً + فاتورة ZATCA بـ VAT 15% وحقولها.
- **F5:** ملخص يومي يُركّب من حالة الشركة ويُرسَل عبر قناة مُهيّأة (وإلا يُسجَّل).
- **F6:** الأفكار دون عتبة الثقة تُوسَم «دون الحد»؛ حلقة cron واحدة تشغّل الدورة كاملة.
