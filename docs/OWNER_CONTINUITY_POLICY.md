# سياسة ترقيم المشاريع واستمرارية الشركة عند غياب المالك

الإصدار: `2026-07-owner-continuity-v1`

## الغرض

تعمل Orvanta تشغيليًا دون اعتماد يومي على وجود المالك، مع بقاء تحديد الاتجاه
الاستراتيجي والالتزامات الجوهرية من صلاحية المالك وحده. لا تمنح هذه السياسة
الوكلاء حق تغيير استراتيجية الشركة، بل تمنحهم حق تنفيذ الخطة المعتمدة وإعادة
النتائج والأدلة.

## الهوية التشغيلية

- يحصل كل مشروع عند إنشائه على رقم متسلسل داخل الشركة وتاريخ مستقلين، مثل
  `10` و`2026-07-16`.
- تحصل كل مهمة وكل إجراء تنفيذي على تسلسل مشترك داخل المشروع بصيغة
  `رقم المشروع/رقم العنصر`، مثل `10/5`، مع تاريخ العنصر.
- لا يُعاد استخدام رقم مشروع أو رقم عنصر محذوف. تحفظ جداول العدّادات آخر رقم
  صدر، وتستخدم أقفال معاملات لمنع التكرار عند الإنشاء المتوازي.
- ترتبط الدراسة، والاعتماد، والمهام، وعمل الوكلاء، والنتائج بالمعرّف الداخلي
  وبالرقم التشغيلي الظاهر للمالك.

## دورة العمل

1. يستقبل المكتب التنفيذي الفكرة ويطلب خطط التسويق والمالية والتشغيل
   والمشتريات والمخاطر.
2. يجمع المكتب الخطط في توصية واحدة، بدائل مقارنة، ميزانية، جدول زمني، شروط
   نجاح وإيقاف، ومهام مرقمة.
3. لا يبدأ التنفيذ الذي يحتاج اعتمادًا قبل تسجيل قرار الاعتماد.
4. بعد الاعتماد ينفذ كل وكيل مهامه داخل اختصاصه وحدوده ويعيد نتيجة قابلة
   للمراجعة.
5. لا تغلق المهمة أثناء غياب المالك من دون دليل أو نتيجة محفوظة.
6. تسجل كل جولة تشغيل أو تفويض أو تصعيد في سجل الاستمرارية.

تحتوي خطة كل مشروع على يوم بداية ويوم تسليم لكل خطوة، وتاريخ إنشاء محفوظ لكل
مشروع ومهمة وإجراء. يبدأ تنفيذ الوكلاء مباشرة بعد اعتماد الخطة، بينما تعمل جولة
التعافي الدورية يوميًا عند `03:15 UTC` (`06:15` بتوقيت الرياض) لاستئناف أي عمل
توقف تقنيًا، ونشر الرسائل المعلقة، وتسجيل المتأخرات والاعتمادات المنتظرة.

## طبقة غياب المالك

يحدد المالك نافذة الغياب والتوجيه الاستراتيجي وحدود الصلاحيات. تدخل السياسة
حيز التنفيذ تلقائيًا عند بداية النافذة وتنتهي تلقائيًا عند نهايتها.

| نوع القرار | التصرف الافتراضي أثناء الغياب |
|---|---|
| عمل داخلي روتيني، مخاطره حتى متوسطة، وقيمته حتى 5,000 ر.س | ينفذه الوكيل ويعيد الدليل |
| قرار تنفيذي غير استراتيجي حتى 25,000 ر.س | يمر بمراجعة وكيل CEO ثم ينفذ |
| تغيير استراتيجية، التزام قانوني، تحويل بنكي، اقتراض، توظيف أو إنهاء، تغيير ملكية | يؤجل ويعود إلى صندوق المالك |
| مخاطرة عالية أو حرجة | يؤجل ويعود إلى صندوق المالك |
| أثر خارجي مثل دفع أو إرسال أو تعاقد | متوقف افتراضيًا، ولا يفتح إلا بقرار صريح من المالك |

يجوز للمالك تعديل الحدود قبل الغياب. جهة الطوارئ البشرية تستقبل التنبيهات ولا
تملك تلقائيًا صلاحية تغيير الاستراتيجية.

## الضوابط التقنية

- كل استعلام وتشغيل مقيد بمعرف الشركة `tenant_id`.
- جداول السياسة والعدادات وسجل الاستمرارية لا تقبل وصول العميل المباشر، وتعمل
  عبر الخادم فقط مع تفعيل RLS.
- بوابة الغياب تعمل داخل دالة حجز الإجراء على الخادم، قبل إنشاء أي أثر خارجي.
- التشغيل الدوري يستأنف أعمال الوكلاء المتوقفة، يحصي المهام المتأخرة
  والاعتمادات المنتظرة، ويسجل نتيجة الجولة.
- حالات التوقف أو انتهاء نافذة الغياب تمنع استمرار التفويض خارج الميثاق.

## دور المالك

وجود المالك مطلوب لتحديد الوجهة، واعتماد الاستراتيجية، وتغيير حدود المخاطرة،
والبت في التصعيدات الجوهرية. وجوده غير مطلوب لمتابعة كل مهمة روتينية أو جمع
نتائج الوكلاء يدويًا.
